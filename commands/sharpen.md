name: sharpen
description: Rewrite a specific application field to score higher on a16z criteria. Use on any answer that scored below 7.5 in /score, or on any draft the founder wants to strengthen before running the full eval.
---

Accept either:
- A field name — pull the current draft from session context
- Pasted raw answer text

Apply the `pitch-narrative` skill's anti-pattern rules and the `eval-answers` skill's scoring dimensions to the single answer.

Rewrite protocol:
1. Score the original on all five dimensions (one-line per dimension)
2. Identify the two lowest-scoring dimensions
3. Rewrite targeting those two dimensions specifically
4. Show: original | rewrite side-by-side
5. Annotate each changed sentence with one-word rationale: `[specificity]` `[velocity]` `[secret]` `[team]` `[voice]`
6. Re-score the rewrite

If the rewrite still scores < 7.5, explain what additional founder input is needed to push it higher. Do not fabricate specifics — if the answer needs a real number, ask for it.

Offer a second pass after founder confirms or corrects the rewrite.
